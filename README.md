# 2024XMCar — STM32F103 循迹小车

基于 STM32F103RCT6 的 PID 循迹小车，使用 MPU6050 陀螺仪做角度闭环控制。

## 硬件

| 模块 | 型号 | 接口 |
|------|------|------|
| 主控芯片 | STM32F103RCT6 | — |
| 陀螺仪 | MPU6050 | I2C PB6/PB7 |
| OLED 屏幕 | SSD1306 128×32 | I2C PB10/PB11 |
| 灰度传感器 | 8 路模拟灰度 + 3-to-8 多路选择器 | GPIO + ADC |
| 电机驱动 | TB6612 | GPIO + PWM |
| 电源 | 7.4V 锂电池 | — |

---

## 引脚详细说明

### OLED 屏幕（SSD1306 128×32）

OLED 通过 I2C 通信，使用两根信号线即可控制。

| OLED 引脚 | STM32 引脚 | 方向 | 说明 |
|----------|-----------|------|------|
| VCC | 3.3V | 供电 | 3.3V 正极 |
| GND | GND | 供电 | 电源地 |
| SCL | **PB10** | I2C 时钟线 | 由 STM32 输出时钟信号 |
| SDA | **PB11** | I2C 数据线 | STM32 和设备双向传输数据 |

代码中创建了独立的 I2C 总线对象 `OLED_WIRE(PB11, PB10)`，以 400kHz 速率运行。

---

### MPU6050 陀螺仪

MPU6050 通过 I2C 通信，返回三轴加速度和三轴角速度，使用内置 DMP（数字运动处理器）计算偏航角（Yaw）。

| MPU6050 引脚 | STM32 引脚 | 方向 | 说明 |
|-------------|-----------|------|------|
| VCC | 3.3V | 供电 | 3.3V 正极 |
| GND | GND | 供电 | 电源地 |
| SCL | **PB6** | I2C 时钟线 | 由 STM32 输出时钟信号 |
| SDA | **PB7** | I2C 数据线 | STM32 和设备双向传输数据 |
| AD0 | GND | 地址选择 | 接地表示使用 I2C 地址 0x68 |
| INT | 不接 | 中断输出 | 本程序使用轮询方式读取，不需要中断 |
| XDA | 不接 | 外部磁力计数据线 | 不使用外部磁力计 |
| XCL | 不接 | 外部磁力计时钟线 | 不使用外部磁力计 |

I2C 速率设置为 100kHz。陀螺仪安装方向为 X 轴朝前。

---

### 8 路灰度传感器（3-to-8 多路选择器 + 模拟输出）

传感器使用一个 3-to-8 多路选择器（类似 74HC4051），通过 3 根数字引脚选择通道 0~7，选中的通道模拟电压从 OUT 引脚输出，由 STM32 的 ADC 引脚读取。

| 传感器引脚 | STM32 引脚 | 方向 | 说明 |
|-----------|-----------|------|------|
| AD0 | **PB13** | 输出 | 通道选择 bit0（最低位） |
| AD1 | **PB14** | 输出 | 通道选择 bit1 |
| AD2 | **PB15** | 输出 | 通道选择 bit2（最高位） |
| OUT | **PA6** | 模拟输入 | 选中的灰度传感器模拟电压输出 |
| VCC | 5V | 供电 | 5V 正极 |
| GND | GND | 供电 | 电源地 |

8 个灰度传感器从左到右排列，对应通道 0 到通道 7。读取时：

1. 设置 AD0/AD1/AD2 选择一个通道（0~7）
2. 等待 20 微秒让信号稳定
3. 用 `analogRead(PA6)` 读取模拟电压（12 位 ADC，范围 0~4095）
4. 循环 8 次读取全部通道

**黑白判断**：
- 白色背景反射强，电压高，ADC 值通常在 2000 以上
- 黑色线条吸收光，电压低，ADC 值通常在 1800 以下
- 阈值：`cal_min × 0.7`，cal_min 是上电时 50 次采样中各通道最大值的全局最小值

读取结果转换为 8 位二进制（bit7 对应最左传感器，bit0 对应最右）：
- 该位为 1 = 传感器下方是黑色（低于阈值）
- 该位为 0 = 传感器下方是白色（高于阈值）

### 双路直流电机（TB6612 驱动）

左电机驱动左后轮，右电机驱动右后轮。每个电机用两根引脚控制方向、一根引脚输出 PWM 控制速度。

**左电机：**

| 电机驱动引脚 | STM32 引脚 | 方向 | 说明 |
|-------------|-----------|------|------|
| AIN1 | **PA3** | 输出 | 方向控制：高电平 = 正转 |
| AIN2 | **PA4** | 输出 | 方向控制：低电平 = 正转 |
| PWMA | **PA2** | 输出 | 速度控制（PWM，TIM2 通道 3） |

**右电机：**

| 电机驱动引脚 | STM32 引脚 | 方向 | 说明 |
|-------------|-----------|------|------|
| BIN1 | **PA0** | 输出 | 方向控制：高电平 = 正转 |
| BIN2 | **PA5** | 输出 | 方向控制：低电平 = 正转 |
| PWMB | **PA1** | 输出 | 速度控制（PWM，TIM2 通道 2） |

速度值的控制逻辑：
- 程序中使用 -50 到 50 的整数表示速度，正数正转，负数反转
- `analogWrite` 映射：`速度值 × 255 / 50`，将速度线性映射到 0~255 的 PWM 占空比
- 实际运行中的速度限制：最小 2，最大 16
- PID 输出限制：循迹 ±10，角度 ±8

---

### 按键

两个按键使用 STM32 内部上拉电阻，按下时引脚读到低电平。

| 按键 | STM32 引脚 | 方向 | 有效电平 | 功能 |
|------|-----------|------|---------|------|
| Key1 | **PC8** | 输入（内部上拉） | 低电平（按下） | 切换运行模式（1→2→3→4→1 循环） |
| Key2 | **PC9** | 输入（内部上拉） | 低电平（按下） | 启动当前选中的模式 |

按键函数 `readKey()` 包含简单的按键消抖（按下等待 20ms，松开等待 20ms）：
- Key1 按下返回 1
- Key2 按下返回 2
- 无按键返回 0

| 外设 | STM32 引脚 | 方向 | 有效电平 | 说明 |

## 使用方法

1. 上电后 OLED 显示 `RDY` 和当前灰度阈值，LED 亮
2. 按 **Key1（模式键，PC8）** 循环切换运行模式（1→2→3→4→1）
3. 按 **Key2（启动键，PC9）** 启动当前选中的模式
4. 运行中 OLED 实时显示：Yaw 角度、左右轮速度、模式、步骤、丢线计数、灰度值

### 运行模式

| 模式 | 编号 | 说明 |
|------|------|------|
| 1 | 0x11 | 角度锁定 0°，在白底上直走，碰到黑线停止并返回选择模式 |
| 2 | 0x12 | 半圆赛道：直走(0°)→黑弧循迹→直走(180°)→黑弧循迹→停止 |
| 3 | 0x13 | 直走(50°)→循迹→直走(130°)→循迹→停止 |
| 4 | 0x14 | 4 圈复杂路线 |

### PID 参数

| 用途 | 比例系数（Kp） | 微分系数（Kd） | 输出限制 |
|------|--------------|--------------|---------|
| 循迹（Track PD） | 2.5 | 3.5 | ±10 |
| 角度（Angle PD） | 0.5 | 3.5 | ±8 |

电机速度范围：2 ~ 16（映射到 PWM 的 0~255）

## OLED 屏幕显示说明

运行中屏幕分三行显示：

| 行 | 内容 | 说明 |
|----|------|------|
| 第 1 行 | `Y:xxx L:xx R:xx` | 当前 Yaw 角度、左轮速度、右轮速度 |
| 第 2 行 | `M:xx S:x lost:xx T/L/X` | 模式编号、步骤编号、丢线次数、跟踪状态（T=跟踪中，L=丢线中，X=弧线退出） |
| 第 3 行 | `I:0xxx mn:xxxx dY:xxx` | 灰度传感器 8 位二进制值、最小灰度 ADC 值、弧线开始以来的 Yaw 变化量 |

## 开发环境

- **框架**：PlatformIO + Arduino
- **编译**：`platformio run`
- **烧录**：`platformio run -t upload`（通过 ST-Link 烧录器）
- **原 Keil 项目**保留在 `User/` 目录下作为参考

## 已知问题

- DMP 初始化在 Arduino Wire 库下偶发失败，当前使用轮询模式读取
- 灰度阈值使用上电自动校准（采样白色表面，取 0.7 倍），如果上电时传感器下方不是白底，阈值可能不准
- ST-Link 烧录偶尔连接失败（硬件接触问题），重新插拔即可
